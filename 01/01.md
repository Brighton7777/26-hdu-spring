### 1
这道题还是有一些思维量在的。
首先是对博弈问题进行分析，手玩了一些比较典的数，如$1$、$2$、$k$、$k+1$、$k+2$等，不难发现只有在石子数是$k+1$的倍数时先手必败，否则先手必胜。
那么问题转化为判断 $a_i\cdot k^{a_i+a_j} + a_j$ 是否为$k+1$的倍数，由于 $2\le a_i,a_j \le k$ ，有 $a_i\cdot k^{a_i+a_j} + a_j \ge k + 1$ ，对 $k+1$ 取模，得到 $a_i \cdot (-1)^{a_i+a_j} + a_j$。
进一步分类讨论：
1. $a_i+a_j$是奇数，则进一步化为$a_j-a_i$，而$2 - k\le a_j-a_i\le k - 2$，且$a_j\neq a_i$，故不能是$k+1$的倍数。
2. $a_i+a_j$是偶数，则判断$a_i+a_j$是否是$k+1$的倍数。

针对上面的第二种情况，我们可以使用根据最大值分治，并通过unordered_map+启发式合并的方式，统计先手必败的次数。
题解中也给出了单调栈+二分的解法，以及笛卡尔树也是可过的，有机会再补吧。

### 2
要使一个序列中任何一个子段之和都不是$k$的倍数，可以通过前缀和转化成判断，在模$k$意义下，有没有两个前缀和具有相同的余数。顺着这个思路，如果序列长度$\le k$时，根据抽屉原理，一定有至少两个前缀和余数相同，即存在一段和是$k$的倍数。而$< k$的情况，因为要求字典序最小的序列，那么只要从$1$开始构造，若填入当前数后的前缀和余数和另一前缀和相同，则尝试填入下一个数，不论如何，总能构造出一个合法序列。

### 3
这道题的第一想法是DP，但逐字符地进行，对于回文串来说并不好维护。
首先考虑拼成的回文串$p=s+t$，$s$是$S$的子串，$t$是$T$的子串，不妨设$|s|\ge|t|$不难发现$t_{\text{reverse}}$一定是$s$的一个前缀，并且如果将$s$拆分为$t_{\text{reverse}}+q$，显然$q$也是一个回文串。
因此我们可以将最后拼成的回文串分为三部分考虑，即$p=s+q+t$，其中$s=t_{\text{reverse}}$，而$q$则是紧接着$s$后面或$t$前面的一个回文串。我们只需要找到所有匹配的$s$和$t$，再累加相应的$q$的个数就行。$S$中以位置$i$为左端点的回文串个数$l_i$，与$T$中以$j$为结束位置的回文串个数$r_j$，可以通过两次Manacher预处理得到。枚举$s$，$t$匹配，即枚举$S$中位置$i$，$T$中位置$j$和长度$len$，复杂度为$O(n^3)$。
考虑如何优化，对于$S$中的$i$位置向后延伸，和$t$中的$j$位置向前延伸，二者可以匹配的最大长度是确定的，这个可以通过字符串hash+二分得到。而知道最大长度$len$之后，在以$i+1\sim i+len+1$为左端点的$S$中的回文串一定存在一种合法的拼接方案，且在$i$，$j$确定的情况下，也只有一种，因此直接对以$i+1\sim i+len+1$为左端点的$S$中的回文串个数求和即可，这可以用前缀和优化。而$T$中的回文串同理。总复杂度优化为$O(n^2\log n)$。
复杂度还能进一步通过dp优化到$O(n^2)$，后面再补吧。

### 4
这道题整体思维难度上来说是不高的。
第一反应就是根据因子的奇偶个数生成01串，两个数相乘就相当于两个01串相异或。可以维护前缀异或和，在枚举单侧端点，将端点的前缀和异或上$x$之后，通过$\text{map}$查询匹配的另一端点个数。
那么如何保存这些01串呢？用bitset吗？
虽然bitset是按位存储的，但可能会有大约$8e4$个质因数，那么一个bitset就需要$8e4 bits = 1e4 Bytes = 10 KB$，要想为$5e5$个数都开一个bitset大约需要$5e6KB = 500 MB$，超出$64MB$的限制。
因此需要对01串这一状态进一步压缩，这里使用$\text{hash}$。对每一个质因数都随机分配一个大的正整数作为对应的$\text{hash}$值，而因为这道题重点在异或上，那么对于一个整数$t$，我们只需要让$t$中所有个数为奇数的质因数的$\text{hash}$值异或起来，作为$t$的$\text{hash}$，这样也天然支持对两个数“相乘”，即相异或。由此，我们就将一个整数质因数分解转化为一个01串，再进一步压缩到一个无符号整数了，保存序列状态所占用的内存为$5e5\times8 Bytes = 4 MB$。
但到这里其实还过不了，因为得到$\text{hash}$的过程相当于对所有质因数做一次分解太慢了。对此可以改用欧拉筛，进一步得到最小质因数表，以加快得到$\text{hash}$的过程，即可通过本题。

### 7
一道不是很难的**树形dp**。
因为沙子只能向根流动，“天然”就适合用树形dp维护。只需要找到每个子树分别能到达的最大的最小沙子数，取这些数的最小值，作为当前根的期望值，有两种情况：
1. 所有子结点将多余的沙子向上传递到根之后，此时根拥有的沙子$\le$期望值$cnt_{min}$，那么期望值合法。
2. 同样上传多余节点，但根沙子数$<$期望值，考虑从所有子结点各抽出同样多的沙子$cnt_{in}$，作为根沙子的补充，那么结果变为$\max_{cnt_{int}} \min(cnt_{root} + cnt_{in} \times size_{son}, cnt_{min} - cnt_{in})$。两侧分别是单调随$cnt_{in}$递增和递减的一次函数，只需要令两侧相等，便可找到两个临界点，取更大即可。

情况2有一点细节需要思考，整体难度不大。
值得注意的一点是，我的习惯判断叶节点的方法是
```cpp
void dfs(int x){
    if(edgs[x].size() == 1) {
        // is leaf node
    }
}
```
但这种情况会有一种可能，就是会把只有一条边的根当作叶节点，因此要对根进行特判，即`if(edgs[x].size() == 1 && fa != 0)`，或者使用其他方法。

这道题成了重灾区。
第一想法是，用并查集维护所有“相关”节点的最大公约数，但打完了发现样例过不了，细想知道，一群数的最大公约数，实际上根本代替不了两个数之间的最小公约数，两个数的最小公约数比一群数限制更少。
实际上，并不用维护数与数之间的关系，而是通过两两之间最小公约数的限制，**直接确定每个数的最小可能组成**，进而判断询问中两个数的最大公约数即可。

### 9
想要解决这道题好像需要一个关键定理——**Steinitz引理**。
即如果有一组二维向量的和为0，且每个向量都落在关于原点中心对称的凸集$B$，那么存在一种重排，使得这个向量序列的所有前缀和都落在$2B$中。
可以非严谨的“证明”引理的正确性，就是既然所有向量都在$B$中，要使最终结果为$0$，当我向一个方向走了一步，最后总会有向着相反方向的向量把我拉回到原点附近，那么只要对向量序列进行合理的排列，就能保证我不离原点太远。
既然所有状态都压缩在$2B$中了，使用BFS遍历所有状态并不难。
这道题与[帆软杯专业组K题](http://eoj.seucpc.com/contest/68/problem/11)有一定相似之处。K题也可以用这道题的方法过，但是在极限复杂度会变成$O(T \times m \times |B|)=O(1e8)$，加了个卡时才过。
